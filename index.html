Names & Binding in JavaScript

A comprehensive guide to identifiers, memory management, binding times, and scope based on your provided documentation.

Part 01

Name Definition & Syntax

Naming Rules

Name is a string of characters used to identify an entity. Names must start with:

  • Letters (a-z or A-Z)
  • Underscore (_)
  • Dollar sign ($)

After the start, they can continue with:

  • Letters, Digits (0-9), Underscore, Dollar sign

Restrictions

A name cannot start with numbers/digits!

Names cannot include special characters: -, @, #, !, ?, *, or space.

valid_names.js
let carName1 = "Volvo XC60";
let carName2 = 'Volvo XC60';

let _lastName = "Johnson";
let _x = 2;
let _100 = 5;

let $ = "Hello World";
let $$$ = 2;
let $myMoney = 5;
errors.js
let 1name;   // error, start with digit not allow
let name 1;  // error, space not allow
let name-1;  // error, hyphen not allow

Design Issues & Keywords

Case Sensitivity

JavaScript is case-sensitive. Upper and lower case letters are different attributes.

let lastName = "Doe";
let lastname = "Peterson";

Reserved Words

Words "reserved from use" for the future. Cannot be used as variable names.

Examples: break, case, class, const, null.

Keywords

Words defined by the language syntax with special meaning.

Examples: if, else, function, return.

Part 02

Address & Memory Management

Memory Address

In JavaScript, every variable has its own memory address internally, but it does not expose or allow direct access to the addresses. This ensures safety and prevents accidental damage. JS is a high-level language using automatic memory management.

Garbage Collection

JavaScript uses the Mark-and-Sweep algorithm. It monitors memory allocation and determines which parts are not used.

  • Has reference? Keep it.
  • No reference? Sweep it (remove).

Storage Types

Stack Memory

Primitive Types

Simple data, cannot be changed.

String: "Hello"
Number: 123
Boolean: true
Null / Undefined
Heap Memory

Reference Types

Complex data, holds reference to location.

Object: {name: "A"}
Array: [1, 2, 3]

Value & Type

Value is the content of the memory. The left value is address, right value is content.
Type refers to the kind of value. JS is dynamically typed (type determined by value assigned).

Aliases

When two or more variable names access the same memory location. While possible via pointers/references, it is harmful to readability.

Part 03

Binding Time

A binding is an association between an identifier and a value/type/address. The moment this occurs is the Binding Time.

1. Language Design Time

Fixed when language is defined (e.g., keywords like import, const). Cannot be redefined by programmer.

module_loading.js
import React from 'react';
import Start from './components/Start';
import Quiz from './components/Quiz';
import { DataProvider } from './context/dataContext';

// The keyword 'import' is bound here. You cannot use 'import' as a variable name.

2. Compile Time (Hoisting)

Corresponds to Creation Phase. Engine creates bindings in memory for variables/functions before execution.

hoisting.tsx
(alias) function App(): JSX.Element
export default App;

// 'App' name is bound to memory location during Compile Phase.

3. Link Time

When multiple files/modules are combined. Links imported variables to correct memory locations in exported files.

linking.js
import Start from './components/Start';
import Quiz from './components/Quiz';

// Binding between 'Quiz' here and actual component file established.

4. Run Time (Dynamic)

Most common. Variables bound to values/types when line executes. Dynamic Type Binding.

Implicit Declaration: We use const/let, but type is implied by value (e.g., true = Boolean).
runtime.js
// Memory allocated when this line runs
const [questionIndex, setQuestionIndex] = useState(0);

// Display Controlling States
const [showStart, setShowStart] = useState(true); // Bound to Boolean
const [showQuiz, setShowQuiz] = useState(false);   // Bound to Boolean
Part 04

Storage Binding & Scope

Stack-Dynamic Variables

Created when declaration reached. Deallocated when function ends. (Local variables/params).

const checkAnswer = (event, selected) => {
    if (!selectedAnswer) {
       // 'event' & 'selected' exist ONLY here
       setCorrectAnswer(question.answer);
    }
}

Implicit Heap-Dynamic

Memory allocated on Heap automatically on assignment. (Arrays/Objects). Resizes automatically.

const [quizs, setQuizs] = useState([]); 
// Initialized as empty array on Heap

useEffect(() => {
   fetch('quiz.json')
     .then(res => res.json())
     .then(data => setQuizs(data)) 
     // Heap expands automatically
}, []);

Scope Types

Static Scoping & Referencing Environment

Scope determined by physical nesting. Inner function accesses outer function variables via Scope Chain. 

export const DataProvider = ({children}) => {
    // Outer Scope variable
    const [showStart, setShowStart] = useState(true);

    // Inner Scope
    const startQuiz = () => {
        // Accesses 'setShowStart' from outer DataProvider
        setShowStart(false);
        setShowQuiz(true);
    }
}

Local Scope

Visible only within the unit/block.

const Quiz = () => {
  // LOCAL to Quiz function only
  const { question, quizs } = 
     useContext(DataContext);
}

Global Scope

Visible to entire program/file (top level).

// Declared at top level
const DataContext = createContext({});

export const DataProvider = ...
// Exported = visible elsewhere

Named Constants

const defines a constant reference. You cannot reassign the variable, but if it's an object/array, you can modify contents. 

const checkAnswer = (...) => { ... }
// checkAnswer reference cannot be changed
