Part 01

Names & Syntax Rules

Understanding valid identifiers is the first step to writing error-free JavaScript.

Allowed Patterns

Identifiers must start with a letter, underscore (_), or dollar sign ($). Subsequent characters can include digits.

valid-names.js 
let carName1 = "Volvo XC60";
let _private = 2;
let $price = 99.50;
let $$$ = 1000;

Strict Restrictions

Names cannot start with digits. No special characters like hyphens or spaces are allowed. JavaScript is also case-sensitive.

errors.js 
// ❌ Syntax Errors
let 1name;     // Starts with digit
let my-name;   // Hyphens not allowed
let user name; // Spaces not allowed

// ⚠️ Case Sensitive
lastName !== lastname

Reserved Keywords

Words reserved by the language syntax cannot be used as variable names.

class return const let function if else import + many more
Part 02

Memory & Data Types

JavaScript creates bindings in memory automatically. Understanding where your data lives (Stack vs. Heap) is crucial for performance.

Stack Memory

Stores static data with fixed size. Fast access, simple structure.

Primitive Types
  • Number 100
  • String "Hi"
  • Boolean true
  • Null null

Heap Memory

Stores complex objects and dynamic data. Slower access, flexible size.

Reference Types
  • Object {...}
  • Array [1,2]
  • Function ()=>

Garbage Collection

Automatic memory management. JS uses the "Mark-and-Sweep" algorithm to free up memory.

"If a variable is no longer reachable, sweep it away."

Part 03

Binding Time & Scope

Language Design Time

Rules fixed when the language specification (ECMAScript) was written. You cannot redefine operators or reserved keywords.

module.js 
import React from 'react';
// 'import' is bound to module loading logic.
// It cannot be used as a variable name.

Compile Time (Hoisting)

Before execution, the engine parses code and allocates memory for declarations. This is why you can call a function declared later in the file.

hoisting.js 
export default App;

function App() {
  // The engine knows 'App' exists here
  // before line-by-line execution begins.
}

Run Time Binding

The most common in JS. Variables get their type and value only when the line of code is actually executed.

Dynamic Type Binding
runtime.js 
const [index, setIndex] = useState(0);
// 'index' is bound to Number type because 0 is assigned.

const [isActive, setIsActive] = useState(true);
// 'isActive' is bound to Boolean type.

Static Scoping

Inner functions access outer variables via the physical nesting in the source code.

Local vs Global

Variables inside a function are private. Variables at the top level are module-scoped.

Global: visible everywhere
Local: visible only in block
